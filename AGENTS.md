# 프로젝트 작업 지침

## 프로젝트 구조

이 프로젝트는 작은 정적 랜딩페이지입니다.

- 주요 마크업은 `index.html`에 있습니다.
- 주요 스타일은 `styles.css`에 있습니다.
- 의도적으로 추가하지 않는 한 빌드 단계나 패키지 매니저는 사용하지 않습니다.

## 작업 기준

- 요청받은 페이지나 동작에 집중해서 수정합니다.
- 이 프로젝트에서는 기본 HTML, CSS, 최소한의 JavaScript를 우선 사용합니다.
- 리디자인 요청이 없다면 기존 시각 방향을 유지합니다.
- 명확한 필요가 없다면 외부 의존성, 프레임워크, 생성형 에셋, 빌드 도구를 추가하지 않습니다.
- 일반적인 데스크톱과 모바일 너비에서 텍스트와 레이아웃이 잘 반응하도록 유지합니다.
- 가능한 곳에서는 시맨틱 HTML을 사용합니다.
- CSS는 기존 스타일에 맞춰 섹션이나 컴포넌트 단위로 정리합니다.

## 확인 방법

- 시각적 변경이 있다면 브라우저에서 `index.html`을 확인합니다.
- 데스크톱과 모바일 너비에서 페이지가 잘 동작하는지 확인합니다.
- 의도하지 않은 텍스트 넘침, 요소 겹침, 깨진 에셋 참조가 없는지 확인합니다.

## 응답 기준

- 변경한 내용을 요약하고 수행한 확인 작업을 함께 언급합니다.
- 브라우저 확인을 하지 못했다면 그 사실을 명확히 말합니다.
